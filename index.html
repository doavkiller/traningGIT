полезные команды git

для командной строки(bash)

команда описание
cd + "название папки" перейти в папку
ls + "название папки или путь" просмотр содержимого папки
mkdir + "название папки" создать папку
touch + "название файла" + .расширение создать файл данного типа
cat "имя файла и .расширение" прочитать указанный файл
mv "имя файла и .расширение" + "имя папки или путь" перенести файл

Основные команды

команда описание
git init инициализировать деректорию
git add + название файла за индексировать(зафиксировать) изменения
git type + "данные" (с моим алиасом -t) название типа данных
git damp + "хэш" (с моим алиасом -p) перейти по хэшу
git cat-file + "способ перехода" + "данные" перейти по указаному пути или открыть файл
git commit -m "текст комментария" создать коммит индексированных файлов
git status текущие состояние данной деректории
git tag "название версии например v1" сохроняет текущию версию файла и присваивает ей имя
git mv "имя и .расширение файла" + "папка или путь до папки куда необходимо перенести файл" перетаскивает файл в другую папку

Отображение комиттов (подробнее)

команда описание
git log стандартное отображение
git log --pretty=oneline отобразить в 1 линию
git log --pretty=oneline --max-count=2 отобразить последние 2 коммита
git log --pretty=oneline --since='5 minutes ago' отобразить коммиты за последние 5 минут
git log --pretty=oneline --until='5 minutes ago' отобразить все коммиты кроме созданных за последние 5 минут
git log --pretty=oneline --author="имя автора" отобразить все коммиты указанного автора
git log --pretty=oneline --all отобразить все коммиты

Отмена изменений

команда описание
git checkout "указать к чему вернуться" Например
1. git checkout "имя файла(ов)"
2. git checkout "хэш файла(ов)"
3. git checkout "тэг файла(ов)" 		вернуться к указаному сохраненому состоянию
1. возвращает файл к последнему коммиту
2. возвращает файл к выбранному коммиту в соответствие с указанным хэшем
3. возвращает к указанной версии
git reset HEAD "название файла" сбрасывает добавленные но еще не закомиченные изменения
git reset --hard "хэш или тэг" удаляет указанный коммит из ветки и из ее истории для полной очистки необходимо еще удалить тег сборшиком муссора git tag -d "указанный тег"
git revert HEAD или хэш коммита удаляет изменения указанного коммита

работа с ответвлениями

команда описание
^ родитель
git branch + "имя ветки" (с моим алиасом -b) создать отвевление
git branch ( -a ) получить информацию о ответвлениях в случае с клонированными репозиториями надо добавить -a
git checkout + "имя ветки" переход к указаное ветке
git merge "название ветки с какой происходит слияние" слияние веток
git rebase "название ветки с какой происходит преобразование" преобразование веток(в отличие от слияния добавляет коммиты)

Клонирование репозиториев

команда описание
git clone "название или адрес репозитория" клонирование репозитория
git remote Получить имя репозитория
git remote show "имя репозитория" получить информацию о репозитории с указаным именем
git fetch получить изменения из главного репозитория (данная команда не сливает их) для обьединения необходимо слияние git merge "название ветки для слияния например origin/master"
git pull получить изменения из главного репозитория и сразу выполнить слияние
git remote add shared "путь к репозиторию.git" создание удалленого репозитория
git push shared "название ветки" отправить изменения в удалленный репозиторий
git remote add shared "путь к репозиторию.git"
git branch --track shared master
git pull shared master 		получить изменения из удалленого репозитория
